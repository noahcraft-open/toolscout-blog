Base64 encoding is everywhere in web development. Email attachments, data URIs, JWT tokens, API authentication headers — they all use Base64. If you've ever seen a string ending in `==` and wondered what it was, that's Base64.

A good online Base64 encoder/decoder saves you from writing one-off scripts every time you need to encode or decode something. Here are the best free options.

## What Is Base64 Encoding?

Base64 is a binary-to-text encoding scheme that converts binary data (bytes) into ASCII text using 64 characters: `A-Z`, `a-z`, `0-9`, `+`, `/`, and `=` for padding.

**Why 64?** Because 64 = 2^6, which means each Base64 character represents exactly 6 bits. Three bytes (24 bits) become four Base64 characters.

### Example
```
Original text: Hello, World!
Base64 encoded: SGVsbG8sIFdvcmxkIQ==
```

The `==` at the end is padding — Base64 always outputs in multiples of 4 characters.

## When Do You Need Base64?

### Common Use Cases

**1. Email Attachments**  
MIME (the email format standard) uses Base64 to encode binary attachments (images, PDFs) as plain text that can be transmitted through email systems.

**2. Data URIs**  
Embed images directly in HTML or CSS without separate file requests:
```html
<img src="data:image/png;base64,iVBORw0KGgoAAAANSUhEUg..." />
```

**3. JWT Tokens**  
JSON Web Tokens use Base64URL (a variant) to encode the header and payload sections:
```
eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJzdWIiOiIxMjM0NTY3ODkwIn0.dozjgNryP4J3jVmNHl0w5N_XgL0n3I9PlFUP0THsR8U
```
Each segment separated by `.` is Base64URL encoded.

**4. HTTP Basic Authentication**  
The `Authorization: Basic` header uses Base64 to encode `username:password`:
```
Authorization: Basic dXNlcjpwYXNzd29yZA==
```
(Note: Base64 is not encryption — anyone can decode this. Always use HTTPS.)

**5. Storing Binary Data in JSON/XML**  
JSON and XML are text formats. Storing binary data (images, files) requires encoding as Base64.

**6. CSS Font Embedding**  
Web fonts can be embedded directly in CSS as Base64-encoded data URIs to reduce HTTP requests.

## Base64 vs. Base64URL

Standard Base64 uses `+` and `/` characters, which have special meanings in URLs. For URL-safe contexts (JWTs, URL parameters), use **Base64URL**:

| Standard Base64 | Base64URL |
|-----------------|-----------|
| `+` | `-` |
| `/` | `_` |
| `=` (padding) | Omitted |

Most JWT libraries handle this automatically, but if you're manually constructing tokens or URL parameters, make sure you're using the URL-safe variant.

## Decoding a JWT Token

JWTs are three Base64URL-encoded segments separated by dots. To inspect a JWT:

1. Split on `.` to get three segments: `header.payload.signature`
2. Base64URL decode the header and payload
3. The signature requires the secret key to verify — you can't decode it with Base64 alone

Example:
```
Token: eyJhbGciOiJIUzI1NiJ9.eyJ1c2VyIjoiYWxpY2UifQ.abc123
Header decoded: {"alg":"HS256"}
Payload decoded: {"user":"alice"}
```

**Security note**: Never paste real production JWTs into online tools. Decode them locally or use your browser's DevTools console:
```javascript
atob('eyJ1c2VyIjoiYWxpY2UifQ')
// → '{"user":"alice"}'
```

## Common Pitfalls

### Character Encoding Issues
Base64 encodes bytes, not characters. Non-ASCII characters (Japanese, emoji, etc.) must be UTF-8 encoded first:
```javascript
// Browser
btoa(unescape(encodeURIComponent('こんにちは')))
// → '44GT44KT44Gr44Gh44Gv'
```

### Line Breaks in MIME Base64
Email MIME Base64 inserts line breaks every 76 characters. Standard Base64 for web use does not. Most tools support both, but check which you need.

### Padding
Base64 output length is always a multiple of 4. Padding `=` characters are added to reach the multiple. Some implementations (especially Base64URL) omit padding — both are valid depending on context.
